Middleware & Auth

  • JWT via jose (sign + verify)
  • Cookie: admin_token (httpOnly)
  • Protege: /admin/* y /api/formularios, /api/stats, /api/config
  • Excluye: /admin/login, /api/auth/*, /api/upload*
  • Redirect a /admin/login si no autenticado
  • 401 JSON para APIs protegidas sin token

CAPA 4 — SERVICIOS EXTERNOS

Supabase

ACTIVOFree Tier
PostgreSQL - 3 Tablas
formulariosid, nombre, email, tel, material, mensaje, archivos (JSONB), estado, nextcloud_path, synced, email_enviado, timestamps
configuracionclave (unique), valor — key-value store
email_logformulario_id (FK), destinatario, asunto, estado, error, proveedor
Storage Bucket: formularios-archivos (publico)
RLS Public insert, service role read/update

Resend

ACTIVOEmail API
  • Email HTML con datos del formulario
  • Links a Admin Panel + NextCloud
  • From: configurable via BD
  • To: configurable (notification_email)
  • Free tier: 100/dia, 3,000/mes
  • Log completo en tabla email_log
  • Tracking uso en dashboard admin

NextCloud (Local)

ACTIVOprintup.internos
  • Servidor local (red interna)
  • WebDAV API (MKCOL, PUT, DELETE, PROPFIND)
  • Carpeta: /PrintUp/Pedidos_Nuevos
  • Auth: Basic (user + app password)
  • Recibe archivos via sync-worker
  • Link directo desde email y admin
  • Almacenamiento definitivo de archivos
Sync Worker (cada 30s)
CAPA 5 — SYNC WORKER (Docker Container)

sync-worker/sync.js

CONSTRUIDO Docker Node.js
  • Poll cada 30s: Supabase formularios WHERE nextcloud_synced = false
  • Download archivos desde Supabase Storage (public URLs)
  • MKCOL: crea carpeta en NextCloud (recursivo)
  • PUT: sube cada archivo via WebDAV
  • DELETE: elimina de Supabase Storage post-sync
  • UPDATE: marca nextcloud_synced = true
  • Retry parcial: si falla un archivo, reintenta proximo ciclo
  • Test de conexion NextCloud al arrancar

Flujo Completo del Formulario

1 Cliente llena formulario y sube archivos
2 Browser obtiene signed URL del API
3 Browser sube directo a Supabase Storage
4 API guarda metadata en PostgreSQL
5 API envia email via Resend al admin
6 Sync Worker detecta nuevo (cada 30s)
7 Worker descarga → sube a NextCloud → borra de Supabase
8 Admin revisa en panel, cambia estado
